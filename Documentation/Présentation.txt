- Méthode 
Projet sous format Scrum avec l'utilisation de Trello pour la création de User Story,
l'utilisation de GIT pour le versionning, passage à GIT FLOW au millieu de projet afin
de pratiquer le fonctionnement de cette méthode 

- Problème rencontré
Méthodologie Scrum, départ du projet difficile suite au non connaissance de Scrum 
ex. Création des User Story  

Pour un Product BurnDown les point sont considéré comme consumé
le product owner les à mis comme "Done
MAIS 
Lorsque c'est un sprint BurnDown cette manière de faire n'est pas possible nous avons donc
mis les points consumé lorsque la User Story passait en "To be reviewe


BurnDown Chart -> à quel moment doit-on consumer les points -> résolution :
-> Product Burndown, dès que le product owner l'a défini comme "Done"
-> Sprint Burndown, dès que la User Story est conisérée comme "To be reviewed" par son créateur

GIT
Difficulté avec GitKraken au départ du projet, début mal organisé -> a force de l'utiliser

Passage à GIT FLOW 
Tout d'abord, le passage c'est fait sur le repository du projet normal, mais beaucoup de bugs
on survenu lors du changement de méthode GIT à GIT FLOW -> Recréatio d'un repository avec 
la méthode GIT FLOW dès le départ, beaucoup plus ordonné suite à la création du nouveau dépot

Projet 
Compréhension du projet au départ non-aquise, mais suite à plusieurs relecture du cahier des 
charge commencement de la création des parties PHP du site

- Améliorations envisageables
Prendre plus de temps au départ du projet à exploiter les outils donnés
